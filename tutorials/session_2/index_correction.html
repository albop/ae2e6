Utilisation des fichiers mod Dynare.

Téléchargez le fichier mod rbc.

Code 
from dyno import dynare

Ouvrez le modèle RBC et résolvez-le. Corrigez les erreurs du fichier mod s’il y en a.

Il y a plusieurs choses à corriger dans le fichier mod :

  • Une erreur de syntaxe ligne 17: il faut supprimer le signe ^ à la fin de la ligne.
  • subtil: une erreur de timing dans la fonction de production ligne ligne 20. Comme le capital utilisé dans la production est prédéterminé à la date t, il faut remplacer k par k(-1).
  • la fonction check renvoie une erreur car le steady-state ne satisfait pas les équations. On peut rajouter steady; avant check; pour recalculer l’état stationnaire.
  • on peut ensuite décommenter la ligne stoch_simul; pour calculer la solution du modèle. Notons que le mode émulation dynare, accepte l’option periods=100 pour changer l’horizon de simulation.
Code 
report = dynare("rbc_fixed.mod")

Interprétez l’effet d’un choc de productivité. Comment dépend-il de la persistance du choc ?

On voit que le choc de productivité augmente immédiatement la produciton l’investissement et la consommation. On remarque que la consommation met plus de temps à attendre son maximum: dans les premières périodes,il faut en effet épargner pour pouvoir investir et profiter mieux du choc de la peristance du choc. Cela crée donc un arbitrage initial entre consommer le gain de productivité et l’investir pour en profiter plus dans le futur.

En changeant la valeur dans le modèle du taux de dépréciation du capital, on peut voir un changement dans le forme du profil de consommation.

# Utilisation des fichiers mod Dynare.

Téléchargez le fichier mod [rbc](rbc.mod).

```{python}
from dyno import dynare
```

__Ouvrez le modèle RBC et résolvez-le. Corrigez les erreurs du fichier mod s'il y en a.__

Il y a plusieurs choses à corriger dans le fichier mod :

- Une erreur de syntaxe ligne 17: il faut supprimer le signe `^` à la fin de la ligne.
- *subtil*: une erreur de timing dans la fonction de production ligne ligne 20. Comme le capital utilisé dans la production est prédéterminé à la date t, il faut remplacer `k` par `k(-1)`.
- la fonction check renvoie une erreur car le steady-state ne satisfait pas les équations. On peut rajouter `steady;` avant `check;` pour recalculer l'état stationnaire.
- on peut ensuite décommenter la ligne `stoch_simul;` pour calculer la solution du modèle. Notons que le mode émulation dynare, accepte l'option `periods=100` pour changer l'horizon de simulation.

```{python}
report = dynare("rbc_fixed.mod")
```
